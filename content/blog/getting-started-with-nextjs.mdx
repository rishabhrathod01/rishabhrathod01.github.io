# Getting Started with Next.js 15: A Comprehensive Guide

Next.js has revolutionized the way we build React applications, and with version 15, it's better than ever. In this guide, we'll explore the key features and best practices for building modern web applications.

## Why Next.js?

Next.js provides an incredible developer experience with features that just work out of the box:

- **Server-Side Rendering (SSR)**: Improve SEO and initial page load times
- **Static Site Generation (SSG)**: Build lightning-fast static websites
- **API Routes**: Create backend endpoints without a separate server
- **Image Optimization**: Automatic image optimization and lazy loading
- **Built-in CSS Support**: Support for CSS Modules, Sass, and CSS-in-JS

## Setting Up Your First Project

Getting started with Next.js is incredibly simple. Open your terminal and run:

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

This will create a new Next.js application with all the necessary configuration files and start the development server on `http://localhost:3000`.

## Understanding the App Router

Next.js 15 uses the App Router by default, which introduces some powerful concepts:

### Server Components

By default, all components in the `app` directory are Server Components. This means they run on the server and send only the rendered HTML to the client.

```tsx
// app/page.tsx
export default function HomePage() {
  return (
    <div>
      <h1>Welcome to Next.js</h1>
      <p>This is a Server Component by default!</p>
    </div>
  )
}
```

### Client Components

When you need interactivity, use the `"use client"` directive:

```tsx
"use client"

import { useState } from "react"

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  )
}
```

## Data Fetching Made Easy

Next.js 15 simplifies data fetching with async Server Components:

```tsx
async function getData() {
  const res = await fetch('https://api.example.com/data')
  return res.json()
}

export default async function Page() {
  const data = await getData()

  return (
    <div>
      <h1>{data.title}</h1>
      <p>{data.description}</p>
    </div>
  )
}
```

## Metadata API for SEO

Next.js 15 provides a powerful Metadata API for SEO optimization:

```tsx
import { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'My Page Title',
  description: 'This is my page description',
  openGraph: {
    title: 'My Page Title',
    description: 'This is my page description',
    images: ['/og-image.jpg'],
  },
}
```

## Best Practices

Here are some best practices I've learned while building with Next.js:

1. **Use Server Components by default**: Only add `"use client"` when you need interactivity
2. **Optimize Images**: Always use the `next/image` component for automatic optimization
3. **Leverage Static Generation**: Use SSG whenever possible for better performance
4. **Code Splitting**: Next.js automatically code-splits your application
5. **Environment Variables**: Use `.env.local` for sensitive data

## Performance Optimization

Next.js provides several built-in optimization features:

- **Automatic Code Splitting**: Each page only loads the JavaScript it needs
- **Prefetching**: Links are automatically prefetched in the viewport
- **Image Optimization**: Built-in image optimization with WebP support
- **Font Optimization**: Automatic font optimization with `next/font`
